Introduction to TypeScript

TypeScript is a strongly typed programming language that builds on JavaScript by adding static type definitions. It is developed and maintained by Microsoft and serves as a superset of JavaScript, meaning all JavaScript code is valid TypeScript.

Why Use TypeScript?

  • Static Typing: Detect errors during development rather than runtime.
  • Improved Tooling: Better IntelliSense, autocompletion, and refactoring in IDEs.
  • Enhanced Code Quality: Type definitions make your code easier to read and maintain.
  • Compatibility: TypeScript compiles to plain JavaScript, making it compatible with any environment running JavaScript.

Who is it for?

TypeScript is ideal for developers working on medium to large-scale applications, especially those requiring long-term maintainability.

History of TypeScript
  • Origins: TypeScript was first introduced by Microsoft in October 2012. Anders Hejlsberg, the creator of C#, was a key figure in its development.
  • Purpose: It was designed to address JavaScript's lack of type safety and improve developer productivity for large codebases.
  • Adoption: Over the years, TypeScript has grown in popularity and is widely used in frameworks like Angular, React, and Vue.
  • Current Version: The language continues to evolve with new features and updates being added regularly.
Getting Started with TypeScript

Prerequisites

  • Familiarity with JavaScript basics.
  • A code editor like Visual Studio Code.

Step-by-Step Setup

  1. Install Node.js:
    • Download and install Node.js
    • Verify installation:
      • node -v npm -v
  2. Install TypeScript:
    • Install the TypeScript compiler globally using npm:
      • npm install -g typescript
    • Verify installation:
      • tsc -v
  3. Create a TypeScript File:
    • Create a new .ts file, e.g., example.ts.
    • Add some TypeScript code:
      • let message: string = "Hello, TypeScript!"; console.log(message);
  4. Compile TypeScript to JavaScript:
    • Use the TypeScript compiler to transpile the .ts file to a .js file:
      • tsc example.ts
    • Run the JavaScript file:
      • node example.js
  5. Initialize a Project (Optional) :
    • Create a tsconfig.json file for project-level configuration:
      • tsc --init
Core Features of TypeScript

Static Typing

  • TypeScript enforces type checks at compile time.
    • let age: number = 25; // Valid age = "twenty-five"; // Error: Type 'string' is not assignable to type 'number'

Interfaces

  • Define custom types with interfaces.
    • interface User { name: string; age: number; } const user: User = { name: "Alice", age: 30 };

Classes and Inheritance

  • Write object-oriented code with TypeScript.
    • class Animal { constructor(public name: string) {} speak() { console.log(`${this.name} makes a noise.`); } } class Dog extends Animal { speak() { console.log(`${this.name} barks.`); } } const dog = new Dog("Rex"); dog.speak(); // Rex barks.

Type Inference

  • TypeScript can infer types automatically, reducing verbosity.
    • let message = "Hello, World!"; // Inferred as string
Conclusion

TypeScript is an incredibly powerful tool for developers seeking to write scalable, maintainable, and error-resistant code. By building on JavaScript and introducing static typing, interfaces, and advanced tooling, TypeScript bridges the gap between flexibility and safety in modern development. With its increasing popularity and adoption in major frameworks and libraries, TypeScript is becoming a must-learn skill for web developers.

Best Practices

  • Use Type Annotations: Explicitly declare types for better readability.
  • Enable Strict Mode: Add "strict": true in tsconfig.json to enforce best practices.
  • Modular Code: Use ES6 modules for better code organization.
  • Leverage TypeScript Libraries: Use libraries with built-in TypeScript definitions for seamless integration.
